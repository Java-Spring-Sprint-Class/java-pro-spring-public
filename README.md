# Welcome to the Spring Backend Course

Welcome to the Backend Development course using the Spring Framework.

During this course, you will work as a real development team, going through sprints, completing tasks, and gradually building a full-featured backend project.

## Course Goal

By the end of this course, you will learn how to:

* Think like a backend engineer, not just "write code"
* Work with the Spring Framework from Core to Security
* Write clean, testable, and maintainable code
* Collaborate in a team using Git, Pull Requests, and code reviews
* Understand why a system works the way it does

## Course Format

* You will work in teams of 2–4 students
* The course is divided into sprints
* Each sprint:
    * has clear requirements
    * includes automated tests
    * becomes available only when the sprint starts
* After each sprint:
    * a grade is assigned
    * the instructor’s reference solution is published
    * you can review it and pull the changes into your code if needed

## Sprint Workflow

1.  **You receive a new sprint**
2.  **Your team:**
    * analyzes the requirements
    * splits tasks among team members
    * works using Git (branches, pull requests)
3.  **Your code must:**
    * pass all standard tests
    * fully meet the sprint requirements
4.  **After the deadline:**
    * grades are published
    * the instructor’s solution is opened
    * you can compare your implementation or pull updates

Then the next sprint is opened, and the cycle repeats.

## Technology Stack

Throughout the course, you will work with:

* Java 21
* Spring Core (IoC / DI)
* Spring MVC & REST
* Spring Data JPA (Hibernate)
* Exception Handling
* Unit & Integration Testing
* Logging
* Spring Security
* JWT Authentication

> **Important:** dependencies and features are introduced gradually, sprint by sprint.

## Course Rules

* Do not copy code from the internet without understanding it
* Do not commit broken code to main
* Work through Pull Requests
* Read the tests — they are the contract
* Ask why, not only how

## Keep in Mind

* Tests are not your enemy.
* Tests define the system requirements.
* The instructor’s solution is not "the only correct one", but one of many possible engineering solutions.
